# LLM Wiki — 주식 투자 지식 베이스 운영 가이드

> Notion "종목재료정리" DB를 Raw Source로 활용하는 LLM 기반 투자 위키 시스템

---

## 세션 부팅 절차 (매 대화 시작 시 반드시 실행)

> ⚠️ **모든 작업 전 `RULES.md` 를 먼저 읽는다.** 절대 누락 불가 규칙이 정리되어 있다.

LLM은 새 대화가 시작되면 아래 순서대로 위키를 읽고 현재 상태를 파악한 뒤 요청을 처리한다.

```
0. RULES.md            → 절대 규칙 확인 (매 세션 필수)
1. wiki/index.md       → 전체 위키 지도 파악 (어떤 종목/테마가 있는지)
2. wiki/overview.md    → 현재 시장 국면 파악 (시황, 주도 섹터, 수급 흐름)
3. 요청 관련 파일      → themes/[테마명].md 또는 stocks/[종목명].md
4. wiki/log.md         → 최근 10줄 확인 (이전 세션에서 무슨 작업을 했는지)
```

> 5단계를 모두 읽은 뒤 "위키 로드 완료. 현재 국면: [국면], 최근 작업: [요약]" 형태로 보고하고 요청 처리 시작.

---

## 핵심 아이디어

지금 Notion의 "종목재료정리" DB에는 종목명, 관련테마, 키워드, 기사 링크, 등락률, 재료 요약이 축적되어 있습니다. 하지만 이 데이터는 현재 **조회용**으로만 쓰이고 있습니다. 새 기사를 볼 때마다 LLM에게 처음부터 다시 설명해야 하고, 테마 간 연결이나 과거 패턴과의 비교는 머릿속에서만 이루어집니다.

여기서 제안하는 방식은 다릅니다.

LLM이 Notion DB를 **원본 소스(Raw Source)로 읽고**, 그 위에 **별도의 투자 위키를 로컬 파일로 직접 구축·유지**합니다. 새 종목 재료가 들어올 때마다 LLM이 해당 테마 페이지, 대장주 분석 페이지, 수급 흐름 페이지를 자동으로 업데이트합니다. 테마에 수급이 몰리는 순간 — **"지금 대장주는 누구고, 어떻게 매매해야 하는가"** — 를 즉시 답할 수 있는 구조가 됩니다.

**위키는 복리로 쌓이는 투자 두뇌입니다.** 재료·차트·거래량·시황의 연결고리는 이미 만들어져 있고, 과거 패턴과의 모순은 이미 표시되어 있습니다. 종목과 테마를 추가할수록 위키는 더 정교해집니다.

## 현재 Notion DB 구조 (Raw Source)

- **DB 이름**: 종목재료정리
- **접근 방식**: 읽기 전용. **절대 수정 금지.**

주요 컬럼:

| 컬럼 | 내용 |
|------|------|
| **종목명** | 종목 식별자 |
| **관련테마** | 멀티셀렉트 (예: #HBM, #자율주행, #방산 등) |
| **카테고리** | 반도체 / AI·데이터센터 / 바이오 / 방산 등 |
| **키워드** | 핵심 투자 키워드 |
| **등락률 (%)** | 당일 등락률 |
| **오늘 기사 내용** | 당일 주요 기사 본문 및 링크 |
| **종목 재료 요약** | 요약 (LLM이 자동 채울 대상) |
| **링크** | 관련 기사 URL |
| **최근업데이트** | 마지막 업데이트 날짜 |

### 종목 페이지 본문 (종목명 클릭 시)

각 종목의 Notion 페이지 안에는 DB 컬럼 외에 아래 본문 정보가 추가로 포함되어 있다. Ingest 시 반드시 페이지 본문까지 읽어야 한다.

| 항목 | 내용 |
|------|------|
| **홈페이지** | 기업 공식 홈페이지 URL |
| **재무사항** | 매출액, 영업이익, 영업이익률, 순이익, 부채비율, PER, PBR, ROE, 시가총액 등 |
| **추가 중요내용** | 사업 구조, 주요 고객사, 경쟁사, 리스크 요인, 기타 투자 참고 메모 |

> Notion MCP로 페이지 본문 접근 시: DB 컬럼 읽기와 별도로 `notion-fetch` 또는 페이지 ID 직접 조회로 본문 블록을 읽어온다.

## 위키 구조 (로컬 파일)

섹터와 테마는 `themes/` 폴더로 통합. frontmatter의 `category` 필드로 구분.

```
wiki/
├── index.md               # 전체 페이지 목록 + 한줄 요약 (세션 부팅 시 1순위)
├── log.md                 # 시간순 활동 기록 (세션 부팅 시 4순위)
├── overview.md            # 시황, 주도 섹터, 수급 흐름 (세션 부팅 시 2순위)
│
├── themes/                # 테마 + 섹터 통합 관리
│   ├── HBM.md             # category: theme
│   ├── 방산.md            # category: theme
│   ├── 자율주행.md        # category: theme
│   ├── 반도체.md          # category: sector
│   ├── 바이오.md          # category: sector
│   └── _TEMPLATE.md
│
├── stocks/                # 종목별 분석 페이지
│   ├── 삼성전자.md
│   ├── SK하이닉스.md
│   └── _TEMPLATE.md
│
└── analysis/              # 질문·분석으로 생성된 페이지 (저장 트리거 참고)
    └── 스크리닝_YYYY-MM-DD.md
```

### 섹터 vs 테마 구분 기준

| 구분 | 정의 | 예시 | category 값 |
|------|------|------|-------------|
| **섹터** | 산업 분류. 종목들의 넓은 소속 그룹 | 반도체, 바이오, 방산, 화학 | `sector` |
| **테마** | 수급을 끌어당기는 재료 기반 그룹. 섹터를 가로지를 수 있음 | HBM, 자율주행, 휴머노이드 | `theme` |

> 실전 팁: "이 종목이 왜 올랐나?"는 테마 페이지를 보고, "이 업종 전반 흐름은?"은 섹터 페이지를 본다.

## 재료 강도 분류 기준

Ingest 및 스크리닝 시 반드시 아래 기준으로 분류한다.

| 등급 | 정의 | 예시 | 투자 관점 |
|------|------|------|-----------|
| **단기 이슈** | 1~2주 내 소멸 가능한 뉴스성 재료 | 계약 발표, 깜짝 실적, 단발 수주 | 모멘텀 매매. 추격 주의 |
| **중기 모멘텀** | 시황 이슈로 수개월 지속되는 재료 | 전쟁·지정학, 금리 방향 전환, 정책 수혜 | 스윙 매매. 재료 지속성 점검 필요 |
| **구조적 성장** | 산업 패러다임 변화, 다년간 지속 | 반도체 사이클, AI 산업 개화, 고령화 | 중장기 보유. 단기 조정에 흔들리지 않음 |

> 판단이 애매한 경우: 재료가 "뉴스 1개"에서 왔으면 단기, "정책/전쟁/기술 전환"에서 왔으면 중기, "10년 트렌드"와 연결되면 구조적으로 분류.

## 기사 업로드 → 종목 스크리닝 워크플로

### 분석 순서

**Step 1 — 시황 파악**
- 미국 증시 흐름 (나스닥·S&P500·필라델피아 반도체지수)
- 환율·금리·유가 등 매크로 변수
- 코스피·코스닥 전반 수급 방향
- 결론: 공격적 매수 / 선별 매수 / 관망 국면 판단

**Step 2 — 테마 추출**
- 반복 등장하는 키워드 = 현재 시장이 주목하는 테마
- 테마별 기사 밀도(언급 횟수)로 수급 집중도 추정
- `themes/` 페이지와 대조해 신규 테마인지, 기존 테마의 재점화인지 구분

**Step 3 — 종목 스크리닝**

| 기준 | 평가 내용 | 비고 |
|------|-----------|------|
| **재료** | 재료 강도 분류 (위 기준 적용), 재료 신선도, 위키 과거 재료와 비교 | LLM 직접 분석 |
| **거래량** | 기사에서 언급된 거래량 급증 여부, 수급 주체 언급 | 기사 텍스트 기반 |
| **시황** | Step 1 시황과의 정합성 | LLM 직접 분석 |
| **차트** | ⚠️ 직접 분석 불가 → 확인해야 할 패턴 가이드 제공 | 성민님이 직접 확인 |

**Step 4 — 종목 추천 리포트 출력**

```
## 📊 오늘의 시황
[시황 요약 2~3줄]
현재 국면: 선별 매수 / 공격 매수 / 관망

## 🔥 주목 테마 순위
1. [테마명] — 기사 언급 N회 / [신규 테마 or 재점화] / [재료 등급]

## 🎯 매매 후보 종목

### 1순위: [종목명] — [한줄 요약]
- 재료: [내용] / 등급: [단기 이슈 / 중기 모멘텀 / 구조적 성장]
- 거래량: [기사 내 언급 근거]
- 시황 정합성: [현재 시장과 맞는 이유]
- 차트 확인 포인트:
  - [ ] 전고점 돌파 여부 (거래량 동반 필수)
  - [ ] 5일선 / 20일선 정배열 여부
  - [ ] 당일 거래량 20일 평균 대비 N배 이상
  매수 시나리오: 위 조건 중 N개 충족 시
  손절 기준: [지지선] 하향 이탈 시
- 리스크: [주의할 점]

## 📁 위키 업데이트 대상
- 신규 종목 페이지 생성 필요: [종목명]
- 기존 테마 페이지 업데이트 필요: [테마명]
- 신규 테마 페이지 생성 필요: [테마명]
```

## 핵심 운영 방식

### 📥 Ingest (재료 추가 시)

> ⚠️ **Ingest 시작 전 `RULES.md` 규칙 2·3·4 반드시 확인.**

전제조건: Notion MCP 연결 확인 (`종목재료정리` DB 접근 가능 여부 확인)

1. 요청: *"삼성전자 재료 ingest 해줘"*
2. LLM 실행 순서:
   - Notion `종목재료정리` DB에서 해당 종목 DB 컬럼 읽기
   - 해당 종목 Notion 페이지 본문 읽기 (홈페이지, 재무사항, 추가 중요내용, **지난기사 아카이브 + 링크**)
   - 재료 분석 및 강도 분류 (단기 이슈 / 중기 모멘텀 / 구조적 성장)
   - `wiki/stocks/[종목명].md` 업데이트 (재무 섹션 포함)
   - 연관 테마 페이지 업데이트
   - `wiki/log.md` 기록 (overview.md는 밤 9시 스케줄에서 일괄 업데이트)

> 재무사항은 분기 실적 발표 이후에만 변경되므로, 이미 `wiki/stocks/[종목명].md`에 재무 섹션이 존재하면 Notion에서 새 수치가 확인된 경우에만 덮어쓴다. 변경 없으면 건너뜀.

#### ⚠️ Ingest 필수 규칙 (wiki/_errors.md 기반)

**[규칙 1 — 재무 데이터]** Notion 재무사항 표는 wiki에 그대로 옮기지 않는다. `python fetch_finance.py [종목명]` 실행으로 DART OpenAPI에서 자동 수집한다. 미실행 종목은 재무 섹션을 `[⏳ fetch_finance.py 실행 필요]`로 표기한다.

**[규칙 2 — 태그]** Notion 관련테마 멀티셀렉트 값은 필터링 없이 100% 그대로 `tags`, `theme` 필드에 복사한다. ingest 후 Notion 개수 = wiki 개수 반드시 확인.

**[규칙 3 — 재료 타임라인 + 기사 아카이브]** Notion 2개 섹션을 100% 전수 반영한다: ① `추가 중요내용`, ② `지난기사 아카이브 + 링크`. 누락·요약 금지.

타임라인 항목 형식: `- [YYYY-MM-DD] [재료 내용] / [등락률: +X%] / [단기/중기/구조적] / [기사 제목](URL)`
- **등락률**: Notion DB `등락률(%)` 컬럼 값을 해당 날짜 항목에 기재. 없으면 `등락률 미기재` 표기
- **링크**: Notion `링크` 컬럼 및 페이지 본문 URL 100% 보존. 없으면 링크 항목 생략
- **기사 아카이브**: `지난기사 아카이브 + 링크` 섹션은 wiki `## 기사 아카이브`에 전문 그대로 복사 (요약·생략 금지)
- LLM 자체 생성 항목 금지. 출처 불명 항목은 `[출처 미확인]` 표기 후 사용자 확인 요청

#### ✅ Ingest 완료 후 체크리스트

- [ ] 재무 섹션: `python fetch_finance.py [종목명]` 실행 여부 확인 (미실행 시 `[⏳ fetch_finance.py 실행 필요]` 표기)
- [ ] 태그 수: Notion 관련테마 N개 = wiki tags+theme N개 일치 확인
- [ ] 타임라인: 각 항목에 등락률 + 링크 포함 확인 (등락률 없으면 `등락률 미기재`, 링크 없으면 생략)
- [ ] 기사 아카이브: Notion `지난기사 아카이브 + 링크` 100% 전수 복사 확인 (누락·요약 금지)
- [ ] log.md 기록 완료

### 🔍 Query (매매 판단 시)

**테마 수급 분석**
- *"지금 HBM 테마에 수급이 몰리고 있어. 대장주는 누구고 어떻게 매매해야 해?"*
- *"오늘 방산 테마 강세인데, 현재 대장주와 2군 종목 정리해줘"*

**종목 매매 판단**
- *"삼성전자 지금 매수 타이밍인지 재료·차트·거래량 기준으로 분석해줘"*

**과거 패턴 비교**
- *"HBM 테마가 올라왔던 과거 패턴이랑 지금이랑 뭐가 달라?"*

### 🔧 Lint (위키 건강 점검)

주 1회 또는 장 마감 후:
- 서로 모순되는 종목 투자 포인트 없는지
- [stale] 태그 달린 오래된 재료/전망 없는지
- 새로 부각되는 테마인데 위키 페이지 없는지
- Notion DB에 새로 추가됐는데 위키에 반영 안 된 종목 없는지

## 분석 결과 저장 트리거

아래 조건 중 하나라도 해당하면 `wiki/analysis/` 에 **반드시** 저장한다.

## overview.md 업데이트 스케줄

- **업데이트 주기**: 주중(월~금) 밤 9시 (자동 스케줄 `wiki-ingest-weekday-9pm`)
- **업데이트 내용 (5단계 순서)**:
  1. Python ingest 스크립트 실행 (`--no-finance --delete-orphans --enrich`)
  2. 시장 데이터 수집 (WebSearch/WebFetch):
     - 코스피/코스닥 종가 및 등락률
     - 코스피/코스닥 상승·하락·보합 종목 수
     - 기관·외국인·개인 순매수 금액 (억원, 코스피 기준)
     - 나스닥 선물 / 코스피200 야간선물 / 코스닥150 야간선물 등락률
  3. Notion `종목재료정리` DB에서 당일 업데이트 종목 전체 읽기 (테마별 그루핑)
  4. `wiki/overview.md` 업데이트 (지수·수급·선물·활성테마·국면 판단 포함)
  5. `wiki/log.md` 기록 + 사용자에게 요약 보고
- **개별 ingest 시**: overview.md는 건드리지 않음. 밤 9시 스케줄에서 일괄 반영.
