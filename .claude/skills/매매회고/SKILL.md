# 매매회고 분석 오케스트레이터

**목표:** 성민님 실제 매매를 ①IMI 원칙 대비 ②실제 결과(outcomes 잠재수익) 대비로 채점해, **반복되는 실수와 다음 매매 체크리스트**를 도출한다. 칭찬·지적 모두 근거(원칙 ID·숫자)와 함께.

> 원칙: 잔소리가 아니라 **데이터 기반 코칭**. 모든 지적은 IMI 원칙 ID(`손절-1` 등)나 숫자(손익비·MFE)로 뒷받침. 판단은 성민님이 하므로, 단정이 아니라 "이 패턴이 반복된다"를 보여준다.

## Phase 0: 데이터 로드

1. **매매일지 (노션 DB)** — `매매일지` DB
   - 데이터소스: `collection://05c63abb-2ddf-490a-8612-e68ee48130e3` (페이지: https://app.notion.com/p/2a2867949fc74b2f93a4bc6b82486b8d)
   - 행 수집: `notion-search` 에 `data_source_url`(위 collection) 지정해 매매 행(페이지) 목록을 얻고, 각 페이지를 `notion-fetch`(page id)로 조회. (전용 query 도구 미노출 — 이 2단계가 정석)
   - **DB 속성은 `종목`+`날짜` 둘뿐**(단순화됨). 나머지 데이터는 **페이지 본문**에 `매매기록` 스킬이 적은 고정 템플릿으로 들어있음 → 본문에서 파싱: 매수가·매도가·손익률(%)·매매유형(단타/스윙)·상태·수량·보유일·진입사유·청산사유·차트메모, 그리고 기존 `🧠 IMI 분석` 블록.
   - **차트 이미지**: 페이지 본문에 첨부돼 있으면 (presigned URL로 보이면) 바이/셀 지점을 IMI 원칙 대비로 본다. 못 읽으면 본문 `차트메모` 텍스트로 분석하고, 정밀 분석은 "차트를 채팅에 직접 첨부"하도록 요청(채팅 첨부는 확실히 분석 가능).
   - **[예시행]** 또는 `[예시` 로 시작하는 감정/메모 행은 제외(샘플).
   - **청산 거래만 통계에 포함**(보유중은 시황분석 보유점검 담당). 청산 건이 없으면 "청산 매매 기록 없음 — 매매일지 DB에 입력 후 다시" 안내.
   - 기간: 요청에 명시 없으면 **최근 4주**. "이번주"/"이번달" 등은 그에 맞춰.
2. **IMI 원칙** `wiki/원칙_IMI.md` — 채점 기준(원칙 ID 체계).
3. **실제 결과** `_cache/outcomes.json` — `"{종목코드}|{날짜}"` → `{fwd:{t1~t10}, mfe_t5, mae_t5}`. 청산 타이밍 vs 잠재수익(MFE) 비교용. 없으면 해당 비교 생략.
4. **국면 매칭(있으면)** `wiki/analysis/시황_*.md` — 각 매매 날짜의 국면(공격/선별/관망)을 끌어와 국면별 성과 분석.
5. (보조) `wiki/stocks/<종목>.md` frontmatter `theme`/`sector` — 테마별 성과 그루핑.

## Phase 1: 분석 (4축)

### 요약 통계 (청산 거래)
거래수 · 승률 · 평균수익(승) · 평균손실(패) · **손익비**(평균수익/평균손실) · 평균보유일 · 단타/스윙 비율.

### 축1 — 손절·비중 규율 (IMI 손절-*, 비중-*)
- **손절**: 손실 거래의 청산사유·손익률 점검. 큰 손실(예: -10%↓)인데 "버팀/존버/기다림"류 사유 → `[⚠️ 손절-1/2 위반]`. 손절선 지켜 작게 끊었으면 ✅.
- **비중**: `수량×매수가`로 종목 투입금 추정. 한 종목 과다 집중(비중-3), 1차 20%·역피라미드 분할(비중-1/2) 준수 여부 점검(감정/메모 참고).

### 축2 — 진입/청산 타이밍 (IMI 진입-*, 차트-*, 시황-3 + outcomes)
- **진입**: 진입사유 분류 — 눌림/지지/저지바/전일고가 돌파(✅ 원칙 정합) vs 추격/상한가 따라감/돌파 직후/뇌동(`[⚠️ 진입-2·시황-3·심리-1]`). 추격매수 비율 집계.
- **청산**: outcomes의 `mfe_t5`(잠재 최대수익)와 실제 손익률 비교 →
  - 실제 ≪ MFE → "수익을 너무 짧게 끊음"(익절 조급).
  - 손실인데 MAE 깊었다 → "손실을 너무 키움".
  - "수익 짧게 / 손실 길게" 패턴(손익비 < 1) 경고.

### 축3 — IMI 원칙 위반 점검 (전수 + 집계)
- 각 매매를 `원칙_IMI.md` 체크리스트로 대조 → 위반 원칙 ID 수집.
- **반복 위반 Top 3**(빈도순) 집계 — 가장 고쳐야 할 습관.

### 축4 — 국면·테마별 성과
- 매매 날짜 ↔ 시황 국면 매칭 → 국면별(공격/선별/관망) 거래수·승률·평균손익. 어떤 국면에서 약한가.
- 테마별(stocks frontmatter) 거래수·승률·평균손익. 강점/약점 테마.

### 종합
- **개선점 Top 3** (반복 위반·손익비·타이밍 중 임팩트 큰 것)
- **다음 매매 체크리스트** (개선점을 행동 가능한 체크박스로)

## Phase 2: 저장 및 보고

1. `wiki/analysis/매매회고_YYYY-MM-DD.md` 저장 (아래 형식).
2. `wiki/log.md` 1줄: `[YYYY-MM-DD HH:MM] 매매회고 — N건/승률X%/손익비1:X, 반복위반:[원칙ID], 개선:[1줄]`

## 주의사항
- 단정·잔소리 금지. "이 패턴이 N회 반복된다"를 **데이터로** 보여주고 개선 방향만 제시. 판단·실행은 성민님.
- 모든 지적에 IMI 원칙 ID 또는 숫자(손익비·MFE·비중) 근거 첨부.
- outcomes/시황 매칭 안 되는 매매는 해당 축에서 "데이터 없음" 표기, 누락하지 말 것.
- 보유중 거래는 통계 제외(회고는 청산 결과 중심). 보유 현황 점검은 시황분석 몫.
- 칭찬할 것은 칭찬(원칙 잘 지킨 매매) — 강화할 행동을 분명히.
