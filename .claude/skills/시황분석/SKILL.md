# 시황 분석 오케스트레이터

**실행 모드:** 단일 에이전트 (market-analyst)
**산출 1순위:** 장 시작 전 한눈에 보는 **⚡오늘의 한 장 (정보 요약)** — 국면·자금흐름·주목종목 현황(레벨/과열)·보유 현황·주의신호·관련 IMI 원칙.
**원칙:** 처방("사라/팔아라") 아님. **팩트를 깔끔히 보여주고 판단은 성민님이.**

## Phase 0: 컨텍스트 확인

### docx 확인
- 첨부됨 → Phase 1
- 미첨부 → 안내:
  ```
  시황_실행.bat 으로 생성한 워드 파일을 첨부해 주세요.
  파일 위치: 오늘의 시황/시황_YYYY-MM-DD.docx
  ```

### 이전 분석 확인
`wiki/analysis/시황_YYYY-MM-DD.md` 존재 시 → "오늘 분석이 이미 있습니다. 덮어쓸까요?" 확인 후 진행.

## Phase 1: 컨텍스트 + 가격 레벨 + 원칙 로드

읽기/실행 실패해도 계속 진행:

1. **가격 레벨:** `python fetch_levels.py --all` → `_cache/levels_YYYY-MM-DD.json` (관심종목 14일 누적 + 보유종목의 현재가·5/20/60일선·60일전고·20일이격·연속상승·거래량배수·RR참고치). 실패해도 계속. 시나리오 종목이 빠졌으면 `python fetch_levels.py --names "A,B"` 보충.
2. **IMI 원칙** `wiki/원칙_IMI.md` — 종목/보유/시황 정보에 관련 원칙 태그용.
3. **보유종목** `보유종목.csv` (levels에 손익률·손절선 병합). 비어 있으면 보유 현황에 미입력 안내.
4. `wiki/overview.md` — 최근 강세 테마 + 국면 메모
5. `wiki/daily_signals/` 최근 3개 — 재등장·테마 랭킹·거래대금
6. `_cache/outcomes.json` — 종목별 T+N·MFE·MAE 백데이터
7. `_cache/situations.json` — 유사 국면 (없으면 생략)
8. **AIDC 매매레이어 (항상):**
   - 갱신: `python aidc_crosscheck.py` → `python aidc_rotation.py` (실패해도 계속)
   - 읽기: `AIDC_매매교차검증_<최신>.md`, `AIDC_순환트래킹.md`, `AIDC_카탈리스트_캘린더.md` / 참조 `AIDC_밸류체인_2026-06-09.md`
9. **LPDDR·베라루빈(SOCAMM) 레이어 (항상):**
   - 읽기: `wiki/themes/LPDDR_베라루빈.md` (관련도순 밸류체인 + 코드 + 스윙메모)
   - 레벨 보충: `python fetch_levels.py --names "제주반도체,오픈엣지테크놀로지,쓰리에이로직스,에이팩트,엠케이전자,심텍,대덕전자,코리아써키트,티엘비,네오셈" --codes "제주반도체:080220,오픈엣지테크놀로지:394280,쓰리에이로직스:354200,티엘비:356860,네오셈:253590,대덕전자:353200"` (실패해도 계속)

## Phase 2: 분석 실행

`market-analyst` 에이전트 호출 (model: opus).

**전달 컨텍스트:** docx 전체 + Phase 1 로드분(특히 levels JSON, 원칙_IMI.md, 보유종목) + 오늘 날짜.

**에이전트 수행 (정보 제시 — 처방 금지, 자세한 룰은 agent 정의):**
1. **국면(시장 신호)** — 공격적/신중/위험 환경 + 관련 IMI 원칙
2. **자금 흐름** — 어디로 돈이 몰리나, 대장주, 신규/지속/과열, 순환 신호(IMI 수급-1)
3. **주목 종목 현황** — levels 숫자 + [스윙후보]/[단타후보] 태깅 + 과열 정량화 + 관련 IMI 원칙. (매수 시나리오 금지, 팩트·주의점까지만)
4. **보유 현황** — 손익·손절까지 거리·이격·거래량 현황 + IMI 태그 (단정 금지)
5. **유사 국면 참조** (situations 있을 때)
6. **AIDC 점검** (필수) — 그룹국면·🚦등급·🔄로테이션·임박트리거, rs 기준, 모순 표기
7. **LPDDR·베라루빈 트래킹** (필수) — `LPDDR_베라루빈.md` 관련도순 + 당일 레벨(이격·전고거리·거래량) 갱신. 직결(🟢🟡 모듈·후공정·기판)과 테마/ASP(⚪ 제주반도체·오픈엣지) 구분 표기, 선반영·과열 정량화. (베라루빈 양산 2026H2~27 = 재료 선반영 구간 명시)

## Phase 3: 저장 및 보고

1. `wiki/analysis/시황_YYYY-MM-DD.md` 저장 확인 (맨 위 ⚡한 장 포함)
2. `wiki/log.md` 기록 확인
3. 사용자 요약 보고:
   ```
   ✅ 시황 정보 정리 완료 — YYYY-MM-DD
   국면: [공격/신중/위험] 환경 — [한줄]
   🔥 자금: [1위테마 대장], [2위 ...]
   📋 주목: [종목([스윙/단타]) 이격+X%·거래량X배], ...  (팩트, 판단은 직접)
   📌 보유: [현황 요약 / 또는 보유종목.csv 미입력]
   🚦 주의: [과열/모순 1줄]
   AIDC: 그룹[위험선호/회피] / 가열[분야] / 추격금지[종목]
   LPDDR: 직결[에이팩트·기판] / 테마탄력[제주반도체·오픈엣지] / 선반영[종목·이격]
   저장: wiki/analysis/시황_YYYY-MM-DD.md
   ```
   > 매매 복기가 필요하면 `매매회고` 스킬("회고해줘") — 실제 매매일지를 IMI 원칙 대비로 분석.

## 에러 핸들링

| 상황 | 처리 |
|------|------|
| levels JSON 없음/실패 | 레벨 칼럼 `차트 직접확인`, RR 생략. 가능하면 재시도 |
| 원칙_IMI.md 없음 | IMI 태그 생략하고 진행 (또는 "IMI 원칙 업데이트해줘" 안내) |
| 보유종목.csv 비어있음 | 보유 현황 `미입력` 안내, 진행 |
| outcomes.json 없음 | 백데이터 없이 진행, 명시 |
| situations.json 없음 | 유사 국면 생략 |
| daily_signals 없음 | 재등장 분석 생략 |
| AIDC 파일 없음 | 갱신 재시도, 실패 시 직전 파일로 진행 |

## 테스트 시나리오

**정상 흐름:** `시황_2026-06-14.docx` 첨부 + "시황 분석해줘" → fetch_levels --all → IMI원칙/보유종목/overview/daily_signals/outcomes/situations/AIDC 로드 → market-analyst가 국면→자금흐름→주목종목 현황(원익IPS[스윙후보] 이격+52% 과열·시황-3 태그 등)→보유현황→유사국면→AIDC → `시황_2026-06-14.md` 저장(⚡한 장 상단) + log.md → 요약 보고.

**에러 흐름 (docx 미첨부):** 안내 후 첨부 시 진행.
**에러 흐름 (레벨 실패):** 레벨 칼럼 `차트 직접확인`으로 두고 진행.
