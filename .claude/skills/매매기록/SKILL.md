# 매매기록 — 차트 → 노션 매매일지 입력 (통합)

**목표:** 성민님이 채팅에 올린 매매 차트(매수/매도 표시)를 읽고, **노션 매매일지 DB에 모든 필드를 채운 한 건을 생성**하며, IMI 원칙 대비 잘한 점·개선점을 함께 적는다. → 입력과 분석을 한 번에 통합.

> 전제: 노션은 이미지 바이너리 업로드를 지원하지 않음 → **차트 이미지 자체는 자동 첨부 불가.** 데이터·분석은 다 채우고, 실제 스샷은 사용자가 폰에서 `차트` 속성에 붙이도록 안내.

## Phase 0: 차트 확인
- 채팅에 차트 이미지가 첨부됐는지 확인. 없으면: "매매한 차트(매수/매도 표시된) 캡처를 첨부해 주세요."
- 여러 매매가 한 장에 있으면 라운드트립(매수→매도) 단위로 분리, 건별로 처리.

## Phase 1: 차트 판독 (이미지에서 추출)
이미지에서 최대한 읽어낸다:
- **종목명** (차트 상단 타이틀). **종목코드**(보이면 사용 / 없으면 `오늘의 관심종목`·`보유종목.csv`로 해결하거나 아는 경우 채움 / 끝내 불명이면 빈칸 + 확인 요청)
- **매수·매도 마커**(바이/셀 화살표)의 가격과 날짜·시각
- **매매유형 추론**: 장중·1일 내 = `단타`, 수일 보유 = `스윙` (일봉/분봉·마커 간격으로)
- **상태**: 매도 마커 있으면 `청산`, 매수만 있으면 `보유중`
- **차트 구조**: 5일선·20일선 대비 위치, 전고점, 거래량(돌파/눌림), 캔들 형태
- **계산**: 손익률(%) = (매도가−매수가)/매수가×100 (소수1자리) / 보유일 = 마커 날짜 차이

## Phase 2: IMI 원칙 대비 분석
`wiki/원칙_IMI.md` 참조해 매매를 채점:
- **진입 평가**: 부합(✅) / 위반(⚠️) 원칙 ID. (예: 저지바 진입=✅`차트-1`, 5일선 눌림=✅`진입-1` / 추격·2연상 추가양봉=⚠️`진입-6`·`차트-8`)
- **청산 평가**: 전고 저항 익절·트레일링(✅`매도-1/2`) / 손절 규율(`손절-1~3`) / 너무 일찍·늦음.
- **차트메모** (객관적 차트 묘사, 한 줄): 예) "5일선 눌림 지지에서 매수, 전고 저항 도달해 매도. 거래량 동반."
- **감정/메모** (IMI 분석, 잘한점+개선점): 예) "✅ 저지바·눌림 진입(차트-1) 좋음 / ⚠️ 1차에 풀매수 — 비중-1·2 어김. 다음엔 10%부터 분할."

## Phase 3: 누락 필드 처리
차트에 안 보이는 값(수량, 정확한 분할 평단, 진입/청산의 실제 '의도')은:
- 차트로 추론 가능한 건 채우고 **추정치는 "(추정)" 표기**.
- 핵심 누락(특히 **수량**, 매매유형 애매)이 1~2개면 **한 번에 간단히 질문**. 사용자가 "그냥 넣어"라면 추정으로 채우고 표기.
- 진입/청산 사유는 차트 근거로 1차 작성하되, 실제 의도와 다를 수 있으니 "수정할 부분 알려달라" 안내.

## Phase 4: 노션 매매일지 입력
`notion-create-pages` (parent: `data_source_id` = `05c63abb-2ddf-490a-8612-e68ee48130e3`).

DB 속성은 **`종목`(title) + `날짜`(date) 둘뿐**(단순화됨). 나머지 데이터·분석은 모두 **페이지 본문(content)**에 아래 고정 템플릿으로 적는다(매매회고가 파싱하므로 형식 유지):
- properties: `종목`(title) = 종목명, `date:날짜:start` = 청산일(보유중이면 매수일).
- content(본문) 템플릿:
  ```
  ## 📈 [매수가] → [매도가] ([손익률%]) · [단타/스윙] · 보유 [N]일
  - 종목코드: [코드]  |  상태: [청산/보유중]
  - 매수: [가격] ([날짜])  /  매도: [가격] ([날짜])  /  수량: [수량 또는 (추정)/미기재]
  - 진입사유: [차트 근거]
  - 청산사유: [차트 근거]
  - 차트메모: [객관적 차트 묘사 — 5일선·전고·거래량 등]

  ### 🧠 IMI 분석
  - ✅ [부합 — 원칙ID]
  - ⚠️ [개선 — 원칙ID]
  - 다음 매매 체크: [한 줄]

  > 차트 이미지는 이 페이지에 직접 첨부(드래그/업로드)
  ```
- 손익률은 매수/매도로 계산(+12, -4.8 형식). 본문 첫 줄(H2)에 핵심값을 모아 표 없이도 한눈에.

## Phase 5: 보고 및 이미지 안내
1. 생성된 페이지 URL 보고 + 채운 핵심값(종목·매수/매도·손익률·유형) 요약.
2. **이미지 안내**: "차트 이미지는 자동 첨부가 안 되니, 방금 만든 항목을 열어 `차트` 속성에 이 스샷을 폰으로 붙여주세요."
3. 추정·빈칸 필드가 있으면 명시하고 수정 요청.
4. (선택) "(예시행)" 샘플이 아직 있으면 정리 권유.

## 주의사항
- 차트에서 읽은 가격·날짜는 **보이는 그대로**. 안 보이면 추정 표기 또는 질문 — 임의 단정 금지.
- 분석은 IMI 원칙 ID 근거로. 잔소리 아닌 코칭(잘한 점도 명시).
- 한 번에 너무 많이 묻지 말 것 — 차트에서 최대한 뽑고, 정말 필요한 것만 확인. "일단 다 넣고 수정" 선호.
- 누적된 매매 **전체 회고**는 `매매회고` 스킬(별도). 이 스킬은 **한 건 입력**.
