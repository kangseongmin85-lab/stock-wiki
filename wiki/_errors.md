# 🐛 Ingest 오류 대장 (Error Ledger)

> 파일럿 테스트에서 발견된 오류를 기록하고, 재발 방지 규칙을 정리한다.
> LLM은 ingest 전 이 파일을 반드시 확인하고 아래 규칙을 적용한다.

---

## 오류 #1 — 재무 데이터 오염 (Financial Table Corruption)

| 항목 | 내용 |
|------|------|
| **발견일** | 2026-05-03 |
| **대상** | stocks/삼성전자.md |
| **증상** | 매출 컬럼에 "-", "-4.6조" 등 비정상 값 표기. 순이익 행 누락. 재무 섹션 공란 표시. |
| **근본 원인** | Notion 페이지 본문의 재무사항이 표(Table) 형식으로 저장되어 있어, Notion MCP로 읽을 때 델타(변동) 값 또는 빈 셀이 잘못 파싱됨. |
| **재발 방지 규칙** | **Notion 재무사항 표는 절대 그대로 wiki에 옮기지 않는다.** 재무 데이터는 반드시 FnGuide API (또는 동등한 외부 금융 API)에서 직접 조회한다. FnGuide 연동 전까지는 재무 섹션을 `[⏳ 재무 데이터 미연동 — FnGuide API 연동 후 자동 채워짐]`으로 표기한다. |
| **상태** | 🔴 미해결 — FnGuide API 연동 필요 |

## 오류 #2 — 관련테마 태그 누락 (Tag Omission)

| 항목 | 내용 |
|------|------|
| **발견일** | 2026-05-03 |
| **대상** | stocks/삼성전자.md (frontmatter tags, theme 필드) |
| **증상** | Notion 관련테마 6개 중 `#반도체 소재/부품` 누락. wiki에는 5개만 반영. |
| **근본 원인** | LLM이 태그 매핑 시 임의 판단으로 "덜 중요하다고 생각한" 태그를 탈락시킴. |
| **재발 방지 규칙** | **Notion 관련테마 멀티셀렉트 값은 필터링·변환 없이 100% 그대로 wiki에 복사한다.** `tags`, `theme` 필드 모두 Notion 원본 값 기준. 개수가 일치하는지 ingest 후 반드시 확인 (Notion N개 → wiki N개). |
| **상태** | 🟡 진행중 — 삼성전자.md 수정 예정 |

## 오류 #3 — 재료 타임라인 소스 오류 (Timeline Sourcing)

| 항목 | 내용 |
|------|------|
| **발견일** | 2026-05-03 |
| **대상** | stocks/삼성전자.md (최근 재료 타임라인 섹션) |
| **증상** | 타임라인이 LLM의 자체 요약·큐레이션으로 작성됨. Notion에 저장된 실제 기사 아카이브와 다를 수 있음. |
| **근본 원인** | Ingest 시 Notion 페이지 본문의 `추가 중요내용` 섹션과 `지난기사 아카이브 + 링크` 섹션을 정확한 소스로 지정하지 않았음. |
| **재발 방지 규칙** | **재료 타임라인은 반드시 Notion 페이지 본문 2개 섹션에서만 추출한다**: (1) `추가 중요내용` — 종목명 포함 기사 정리, (2) `지난기사 아카이브 + 링크` — 날짜·제목·링크 포함 기사 목록. LLM 독자 추론으로 타임라인 항목을 생성하는 것은 금지. 출처가 없는 항목은 `[출처 미확인]` 표기 후 사용자 확인 요청. |
| **상태** | 🟡 진행중 — 삼성전자.md 타임라인 재작성 예정 |

## 오류 점검 체크리스트 (Ingest 완료 후 실행)

ingest 완료 후 아래를 순서대로 확인한다:

- [ ] 재무 섹션: Notion 표 값 사용 여부 점검 (있으면 즉시 제거 → FnGuide 대기 표시로 대체)
- [ ] 태그 수 검증: Notion 관련테마 개수 vs wiki tags+theme 개수 일치 확인
- [ ] 타임라인 소스: 각 항목에 Notion 섹션 출처 명시 또는 `[출처 미확인]` 표기 확인
- [ ] log.md 기록 완료 여부
