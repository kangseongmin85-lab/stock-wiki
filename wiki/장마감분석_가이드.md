# 장마감 후 시황 분석 가이드 (매일 아침 클로드용)

> 사용자가 매일 아침 일어나서 클로드에게 한 줄 명령하면 클로드가 이 문서를 읽고 정확히 같은 절차로 분석한다.

---

## 매일 아침 한 줄 명령 (사용자가 입력)

```
어제 장마감 후 분석해줘
```

또는 동등 표현: "어제 15시 이후 기사 분석", "장마감 후 시황 분석", "오늘 아침 분석 시작"

---

## 클로드가 따라야 할 절차

### Step 1. 분석 윈도우 확정
- 시작: **어제 15:00 KST** (KRX 정규장 마감 시각)
- 종료: **오늘 05:30 KST** (사용자가 일어나기 직전)
- 윈도우 명시: `YYYY-MM-DD 15:00 ~ YYYY-MM-DD 05:30 KST`

### Step 2. Notion 조회 대상 DB
1. 주간 DB 로테이션이 활성화되어 있다 (`WEEKLY_ROTATION=true`).
2. 현재 주차 DB 제목: `📡 시황 아카이브 YYYY-Www (MM/DD~MM/DD)`
3. 윈도우가 **월요일 06:00 KST 이전**이면 — 어제(일요일)가 이전 주에 속하므로 **이전 주차 DB도 함께** 조회한다.
4. 그 외 평일은 현재 주차 DB 하나만 조회한다.
5. 부모 페이지 ID 는 현재 NOTION_DB_ID 의 parent 에서 자동 조회 가능 (`weekly_db.get_parent_page_id`).
6. 필요한 경우 `python weekly_db.py` 실행으로 현재 주차 DB ID 확인.

### Step 3. Notion 쿼리 필터
```json
{
  "filter": {
    "property": "날짜",
    "date": {
      "on_or_after":  "어제 06:00:00+09:00",
      "on_or_before": "오늘 05:30:00+09:00"
    }
  },
  "sorts": [{"property": "날짜", "direction": "ascending"}]
}
```
- "어제 06:00" 으로 살짝 더 넓게 잡아도 됨 — 장중 후속 보도가 장마감 시각에 걸쳐 들어오는 경우가 있어 절단 방지.
- 페이지 100건씩 has_more 끝까지 페이징.

### Step 4. 카테고리 필터 (중요)
- `카테고리 == "시황분석"` 인 항목은 분석 대상에서 **제외** (이전 분석 결과 자기재귀 방지).
- `중요도 == "낮음"` 은 1차 패스에서 제외, 섹터별 기사 수가 적을 때만 보완으로 포함.

### Step 5. 섹터 분류
다음 8개 섹터로 그룹핑한다. 종목명·테마태그·키워드로 매핑:

| 섹터 | 매핑 키 (테마태그 / 키워드) |
|------|------------------------------|
| 반도체 | 반도체, HBM, 낸드, D램, 파운드리, AI반도체 |
| 바이오/제약 | 바이오, 제약, 임상, FDA, 신약, 항암, NDA, IND |
| 2차전지 | 2차전지, 배터리, 양극재, 음극재, 전해질, 분리막 |
| AI/로봇 | AI, AI/로봇, 인공지능, 로봇, 자율주행, 챗GPT, LLM, 휴머노이드 |
| 방산 | 방산, K방산, 방위산업, 무기수출 |
| 조선 | 조선, LNG선, 수주, 조선업, 해양플랜트 |
| 자동차 | 자동차, 전기차, 수소차, 친환경차 |
| 원자력/에너지 | 원자력, 원전, SMR, 에너지, 신재생 |

매핑되지 않는 기사는 "기타" 로 모아 별도 섹션. 매크로(환율/금리/유가) 는 별도 섹션.

### Step 6. 섹터별 중요 기사 선별 (각 3~5건)
다음 우선순위로 정렬:
1. **중요도 == "높음"** 인 기사 우선
2. 같은 종목·재료가 여러 출처에서 반복 보도된 것 우선 (수급 집중 신호)
3. 1차 재료 (계약/실적/FDA/공시) > 2차 분석 (증권사 코멘트, 외인 동향)
4. 같은 종목 기사가 3건 이상이면 1건만 대표로, 나머지는 "함께 보도" 한줄 요약

각 기사에 대해:
- 한 줄 요약 (50자 이내)
- 원문링크 (Notion 의 `원문링크` 필드)
- 재료 등급 (단기 이슈 / 중기 모멘텀 / 구조적 성장 — `CLAUDE.md` 재료 강도 분류 기준)

### Step 7. 관련주 추출
- 기사 본문 + 제목에서 종목명 추출
- `wiki/stocks/*.md` 파일명 풀과 **반드시 교차 검증** — 매칭된 종목만 출력
- 매칭 안 되는 종목명은 `[?]` 표시 후 제외 (LLM 환각 차단)
- 한 기사당 관련주 최대 5개

### Step 8. 출력 포맷
```
# 장마감 후 시황 분석 YYYY-MM-DD

## 분석 윈도우
어제 15:00 ~ 오늘 05:30 KST | Notion 조회 N건 (시황분석 제외 후)

## 📊 전반 시황
- 어제 코스피/코스닥 종가 (가능하면 보강)
- 미국증시 야간 흐름 (나스닥/S&P/필반)
- 환율·금리·유가 한 줄

## 🏭 섹터별 핵심 기사 + 관련주

### 1. [섹터명] — 기사 N건 / 강도: [단기/중기/구조적 혼합]
- **[기사1 한줄 요약]** (출처) — 관련주: 종목A, 종목B / 등급: 중기 모멘텀 → [원문](URL)
- **[기사2]** ...

(섹터 8개 + 기타 + 매크로 순서로 반복)

## 🎯 오늘 주목 후보 (TOP 5)
시황 정합성 + 재료 강도 + 수급 집중 기준 종합 점수로 종목 5개 선정.
각 종목은 wiki/stocks/[종목명].md 페이지 링크와 함께 표기.

## 📁 저장
- 본 분석은 `wiki/analysis/장마감분석_YYYY-MM-DD.md` 로 저장 (analysis 트리거)
- log.md 에 1줄 기록
```

### Step 9. 저장
- `wiki/analysis/장마감분석_YYYY-MM-DD.md` 신규 작성 (CLAUDE.md 분석 결과 저장 트리거 준수)
- `wiki/log.md` 마지막에 한 줄: `- YYYY-MM-DD HH:MM | 장마감분석 완료 — 시황 X건 분석, TOP5 추출`
- Notion DB 자동 저장은 하지 않음 (사용자가 매일 직접 확인하므로 wiki 만으로 충분)

---

## 클로드가 절대 하지 말 것

- 위키 `stocks/*.md` 풀과 매칭되지 않는 종목명을 추천에 올리지 않는다. (환각 차단)
- 차트 패턴을 단정적으로 분석하지 않는다. CLAUDE.md 의 "차트 확인 포인트" 체크리스트로만 제시.
- "사세요 / 매수하세요" 표현 금지. 반드시 "시나리오 / 진입 조건" 형태.
- 윈도우 밖 기사(어제 15:00 이전, 오늘 05:30 이후)는 포함하지 않는다.
