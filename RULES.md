# RULES.md — 절대 누락 불가 규칙

> CLAUDE.md의 운영 지침 중 **절대 어겨서는 안 되는 항목만** 추출.
> 모든 작업(세션 시작 / Ingest / 분석) 전에 이 파일을 먼저 확인한다.

---

## 1. 세션 부팅 순서 (4단계 전부 실행, 생략 불가)

```
1. wiki/index.md       읽기  → 전체 위키 지도 파악
2. wiki/overview.md    읽기  → 현재 시장 국면 파악
3. 요청 관련 파일      읽기  → stocks/ 또는 themes/ 파일
4. wiki/log.md         읽기  → 최근 10줄 (이전 작업 파악)
```

완료 후 반드시 보고:
> "위키 로드 완료. 현재 국면: [국면], 최근 작업: [요약]"

---

## 2. Ingest — 3대 절대 규칙

### 규칙 A — 재무 데이터

| 해야 할 것 | 하면 안 되는 것 |
|---|---|
| `python fetch_finance.py [종목명]` 실행 | Notion 재무표 그대로 복사 |
| 미실행 시 `[⏳ fetch_finance.py 실행 필요]` 표기 | LLM이 수치를 추정·생성 |

### 규칙 B — 태그

| 해야 할 것 | 하면 안 되는 것 |
|---|---|
| Notion `관련테마` 멀티셀렉트 **100% 전수 복사** | 임의로 필터링·제거 |
| 검증: Notion 개수 = wiki tags+theme 개수 일치 확인 | 일치 확인 생략 |
| 공백 포함 태그는 하이픈으로 치환 (예: `반도체-소재/부품`) | 공백 있는 태그 그대로 사용 |

### 규칙 C — 타임라인 + 기사 아카이브

**타임라인 항목 형식 (이 형식 외 금지):**
```
- [YYYY-MM-DD] [재료 내용] / [등락률: +X%] / [단기/중기/구조적] / [기사 제목](URL)
```
- 등락률 없으면 → `등락률 미기재`
- 링크 없으면 → 링크 항목 생략 (나머지는 유지)

**기사 아카이브 규칙:**
- Notion `지난기사 아카이브 + 링크` 섹션 → `## 기사 아카이브`에 **전문 그대로** 복사
- 요약·생략·재구성 금지. 100% 전수 복사
- LLM 자체 생성 타임라인 항목 금지 — 출처 불명이면 `[출처 미확인]` 표기 후 사용자 확인 요청

---

## 3. Ingest 실행 순서 (이 순서 엄수)

```
1. Notion DB 컬럼 읽기        (종목명, 관련테마, 등락률, 링크 등)
2. Notion 페이지 본문 읽기    (홈페이지, 재무사항, 추가 중요내용, 지난기사 아카이브 + 링크)
3. wiki/stocks/[종목명].md   업데이트
4. 연관 themes/*.md          업데이트
5. wiki/log.md               기록
```

❌ **overview.md는 개별 ingest에서 절대 수정하지 않는다** → 밤 10시 스케줄 전용

---

## 4. Ingest 완료 후 체크리스트 (5개 전부 확인)

- [ ] **재무**: `fetch_finance.py` 실행 완료 (또는 `[⏳]` 표기)
- [ ] **태그 수**: Notion 개수 = wiki 개수 일치
- [ ] **타임라인**: 등락률 + 링크 포함 (없으면 `등락률 미기재` / 링크 생략)
- [ ] **기사 아카이브**: 지난기사 100% 전수 복사 (누락·요약 없음)
- [ ] **log.md**: 기록 완료

---

## 5. 절대 금지 항목

| 금지 행동 | 위반 시 발생하는 문제 |
|---|---|
| Notion DB 수정 | Raw Source 오염 — 복구 불가 |
| LLM 자체 생성 타임라인 항목 | 출처 불명 정보로 매매 오판 |
| Notion 재무표 직접 복사 | DART 공식 수치와 불일치 가능 |
| 단정적 매수/매도 추천 | 반드시 "시나리오" 형태로만 제시 |
| 개별 ingest 시 overview.md 수정 | 밤 10시 스케줄과 충돌 |
| 태그 필터링 | 테마 연결 끊김 → 스크리닝 누락 |

---

## 6. 이상 항목 처리 기준

| 상황 | 처리 방법 |
|---|---|
| 재료가 서로 모순 | `[⚠️ 모순: 내용]` 표기하고 계속 진행 |
| 출처를 확인할 수 없는 항목 | `[출처 미확인]` 표기 후 사용자 확인 요청 |
| 오래된 정보 (stale) | `[stale: YYYY-MM-DD]` 태그 추가 |

## 7. 분석 저장 트리거 (해당 시 반드시 저장)

| 상황 | 파일명 |
|---|---|
| 기사·뉴스 첨부 스크리닝 | `wiki/analysis/스크리닝_YYYY-MM-DD.md` |
| 재료 강도 분석 포함 질문 | `wiki/analysis/[종목명]_재료분석_YYYY-MM-DD.md` |
| 거래량·수급 관련 질문 | `wiki/analysis/[테마명]_수급분석_YYYY-MM-DD.md` |
| 시황 종합 분석 질문 | `wiki/analysis/시황_YYYY-MM-DD.md` |

저장 불필요: 단순 사실 확인 질문 ("섹터가 뭐야?", "정의가 뭐야?" 등)
